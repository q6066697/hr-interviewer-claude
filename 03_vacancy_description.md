# Вакансия: Python-разработчик (Middle)

**Компания:** CodeWave Labs
**Команда:** Core Platform / Integration Services
**Формат:** удалённо (UTC-1..UTC+4)
**Занятость:** полная, 40 часов в неделю
**Вилка:** $2 800 — $4 200 net (обсуждается на следующем этапе)

## О команде

Команда **Integration Services** (7 инженеров, 1 тимлид, 1 QA) отвечает за модуль интеграций WaveFlow с внешними сервисами: Salesforce, HubSpot, Slack, Google Workspace и ещё ~80 коннекторов. Пишем асинхронный бэкенд на FastAPI, много работаем с очередями, ретраями и идемпотентностью.

## Чем предстоит заниматься

- Разрабатывать новые коннекторы к внешним API (REST, GraphQL, webhooks).
- Поддерживать и развивать существующие интеграции.
- Работать над надёжностью: ретраи, circuit breakers, dead-letter queues.
- Участвовать в архитектурных обсуждениях и ADR.
- Проводить код-ревью коллег.
- Участвовать в on-call (1 неделя в 6–8 недель, с доплатой).

## Что ожидаем от кандидата (hard skills)

- **2+ года** коммерческого опыта разработки на Python.
- Уверенное владение **asyncio**, понимание event loop, разницы между sync/async кодом.
- Опыт работы с одним из фреймворков: **FastAPI** (приоритет), Starlette, AIOHTTP. Опыт на Django/Flask — тоже ок, но будет нужна адаптация.
- Опыт работы с **PostgreSQL**: индексы, EXPLAIN, транзакции, базовая оптимизация запросов.
- **SQLAlchemy 2.0** или аналогичный ORM.
- Опыт работы с **очередями** (RabbitMQ / Kafka / Celery) в продакшене.
- Понимание принципов **REST API** и HTTP (методы, коды, идемпотентность).
- Опыт написания **тестов**: pytest, моки, фикстуры.
- **Docker** на уровне уверенного пользователя (писать Dockerfile, понимать слои).
- Git, pull requests, code review.

## Будет плюсом

- Опыт проектирования интеграций с внешними API.
- Опыт работы с Kubernetes (чтение манифестов, kubectl).
- Опыт с observability: Prometheus, Grafana, структурированные логи.
- Знание паттернов: retry with backoff, circuit breaker, saga.
- Английский B1+ (для чтения доков и общения в мультикультурной команде).

## Soft skills

- Самостоятельность — умение декомпозировать задачу и довести её до релиза.
- Ответственность — за результат, а не только за написанный код.
- Коммуникация — умение аргументированно отстаивать решение и принимать чужое.
- Способность работать с размытыми требованиями и задавать правильные вопросы.

1. **Скрининг** — интервью с HR-ассистентом (этот шаг).
2. **Техническое интервью** — 1 час с тимлидом и одним из инженеров.
3. **Тестовое задание** — 4–6 часов, на выходные. Оплачиваемое.
4. **Финальное интервью** — с CTO.
5. **Оффер.**

Общая длительность — 1–2 недели с момента первого контакта.
