# 🤖 TechHire Interviewer — HR-ассистент на базе Claude Projects

Кастомный AI-ассистент для автоматизации первичного интервью с кандидатами на позицию Python-разработчика (Middle). Реализован на платформе **Anthropic Claude** с использованием Projects — нативного механизма кастомизации с базой знаний и инструкциями.

> 🌐 **Демо:** [ссылка на проект в Claude] *(добавить после публикации)*

---

## 🎯 Бизнес-задача

- Снизить нагрузку на HR-команду за счёт автоматизации первичного отбора.
- Стандартизировать интервью: все кандидаты получают одинаковые вопросы и единые критерии оценки.
- Ускорить воронку найма — первичная фильтрация занимает 15–20 минут и не требует времени HR.
- Получать структурированный отчёт, пригодный для передачи тимлиду без переработки.

## 🛠 Как устроено

Ассистент построен на платформе **Claude Projects** (no-code).

- **Системный промпт** с жёсткой структурой из 5 этапов интервью, правилами ведения диалога и шаблоном финального отчёта.
- **База знаний** из 3 документов: компания, вакансия, портрет кандидата. Claude использует их через встроенный Contextual Retrieval для более точных ответов.
- **Автоматическая генерация отчёта** с оценками по шкале 1–5, средним баллом и итоговой рекомендацией по фиксированным правилам.

## 💡 Почему Claude, а не GPTs

- **Бесплатный старт** — до 5 проектов на free-тарифе, без подписки.
- **Сильная работа с длинными промптами и базами знаний** — особенно на русском языке.
- **Платформо-независимая архитектура** — те же файлы за 10 минут переносятся на OpenAI GPTs при необходимости (см. `05_setup_guide.md`, часть 7).

## 🚀 Быстрый старт

1. Откройте [claude.ai](https://claude.ai) → **Projects** → **Create Project**.
2. Вставьте содержимое `01_system_prompt.md` в **Custom instructions**.
3. Загрузите файлы 02–04 в **Project knowledge**.
4. Откройте новый чат, напишите «Начните интервью» — протестируйте.
5. Поделитесь ссылкой на проект с кандидатами.

Подробная инструкция и процесс работы с кандидатами — в `05_setup_guide.md`.

## 🧪 Тестовые сценарии

Проект покрыт тремя сценариями: сильный кандидат, слабый, пограничный. Чек-лист валидации и ожидаемые результаты — в `05_setup_guide.md`, раздел «Тестирование перед запуском».

## 🔐 Работа с персональными данными

Ассистент спроектирован с учётом требований к обработке персональных данных: не запрашивает и не фиксирует чувствительную информацию (паспорт, адрес, финансовые реквизиты). Подробности о GDPR / 152-ФЗ и ограничениях — в `05_setup_guide.md`, часть 6.
