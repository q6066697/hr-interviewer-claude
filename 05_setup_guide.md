# Инструкция: развёртывание и поддержка HR-ассистента в Claude Projects

Этот документ описывает, как развернуть, протестировать и поддерживать HR-ассистента **TechHire Interviewer** на платформе **Claude Projects** (claude.ai).

## Почему Claude Projects

- **Бесплатный старт** — на бесплатном тарифе Claude доступно до 5 проектов с кастомными инструкциями и базой знаний.
- **Лучше работает с длинными промптами** — на платных планах контекстное окно 200 000 токенов, файлы базы знаний обрабатываются через Contextual Retrieval.
- **Отличное качество на русском языке** — Claude стабильно держит тон, формат и структуру в русскоязычных диалогах.
- **Простая публикация** — проектом можно делиться через прямую ссылку, не требуется отдельный «магазин».

## Часть 1. Создание проекта в Claude

### Шаг 1. Создание нового проекта

1. Откройте [https://claude.ai](https://claude.ai) и войдите в аккаунт.
2. В левом сайдбаре найдите раздел **Projects** (значок папки).
3. Нажмите **Create Project** (или **+ New Project**).
4. Заполните:
   - **Name:** `TechHire Interviewer — Python Middle`
   - **Description:** `HR-ассистент для первичного интервью Python-разработчиков уровня Middle. Проводит структурированное интервью и готовит отчёт для HR-команды.`
5. Нажмите **Create**.

### Шаг 2. Настройка инструкций проекта

1. Внутри проекта найдите раздел **Custom instructions** (или **Set project instructions** — название поля зависит от версии интерфейса).
2. Откройте локально файл `01_system_prompt.md`.
3. Скопируйте **всё содержимое файла** (Ctrl+A → Ctrl+C).
4. Вставьте в поле инструкций проекта.
5. Нажмите **Save**.

> 💡 Claude не имеет отдельных полей «Name / Description / Instructions» как в GPTs — инструкции задаются одним большим блоком. Поэтому системный промпт уже включает все нужные разделы.

### Шаг 3. Загрузка базы знаний

1. В правой части страницы проекта найдите раздел **Project knowledge** (или кнопку **+ Add content**).
2. Загрузите три файла через drag-and-drop или кнопку **+**:
   - `02_company_profile.md`
   - `03_vacancy_description.md`
   - `04_ideal_candidate.md`
3. Дождитесь обработки (5–10 секунд на файл — появится индикатор готовности).

**Поддерживаемые форматы:** `.md`, `.txt`, `.pdf`, `.docx`, `.csv`, `.html`. Максимум 30 МБ на файл.

### Шаг 4. Совместное использование проекта

1. В правом верхнем углу проекта найдите кнопку **Share**.
2. Доступные варианты:
   - **Private** — только вы (для внутреннего тестирования).
   - **Share with link** — любой человек по ссылке (подходит для рассылки кандидатам).
3. Скопируйте ссылку — её отправляете кандидату в письме-приглашении.

> ⚠️ **Важно:** на бесплатном и Pro-тарифах совместный доступ работает в режиме «просмотр и чат». Редактировать настройки могут только Team и Enterprise.

## Часть 2. Тестирование перед запуском

Перед тем как отправить ссылку первому кандидату, проверьте ассистента на 3 сценариях.

### Тест 1. «Сильный кандидат»

Отвечайте на все вопросы детально, с примерами из практики, упоминая async/await, EXPLAIN ANALYZE, RabbitMQ, idempotency.

**Ожидаемый итог:** рекомендация «Рекомендовать», средний балл 4+.

### Тест 2. «Слабый кандидат»

Отвечайте общими фразами («ну у нас всё работало», «я писал код»), путайтесь в понятиях.

**Ожидаемый итог:** рекомендация «Не рекомендовать», средний балл ниже 3.0, зафиксированы red flags.

### Тест 3. «Пограничный кандидат»

Сильные ответы по Python, но слабые по архитектуре и базам.

**Ожидаемый итог:** «Рекомендовать с оговорками» с указанием, что именно проверить на техническом интервью.

### Чек-лист проверки

- [ ] Ассистент представляется и информирует о формате перед началом.
- [ ] Задаёт вопросы по одному, а не пакетом.
- [ ] Не подсказывает ответы.
- [ ] Держит нейтральный тон (не хвалит, не критикует в процессе).
- [ ] Задаёт уточняющий вопрос, если ответ слишком общий.
- [ ] В конце генерирует отчёт в указанном формате с таблицами.
- [ ] Оценки в отчёте соответствуют правилам (1–5).
- [ ] Итоговая рекомендация согласована со средним баллом.
- [ ] Ассистент не сохраняет и не переспрашивает чувствительные персональные данные.
- [ ] Ассистент ссылается на данные из базы знаний (упоминает CodeWave Labs, стек, культуру).

## Часть 3. Процесс работы с кандидатом

### Типовой сценарий

1. HR отправляет кандидату письмо со ссылкой на проект и коротким описанием: «Первичное скрининг-интервью в формате чата с Claude, 15–20 минут».
2. Кандидат открывает ссылку, логинится в Claude (бесплатный аккаунт подходит).
3. Внутри проекта кандидат нажимает **Start new chat** и пишет «Начните интервью».
4. По завершении кандидат копирует финальный отчёт и присылает HR-менеджеру.

### Шаблон письма кандидату

```
Здравствуйте, [Имя]!

Благодарим за отклик на позицию Python-разработчик (Middle) в CodeWave Labs.

Первый этап отбора — короткое структурированное интервью в формате
чата с нашим HR-ассистентом на базе Claude. Займёт 15–20 минут,
пройти можно в удобное время в течение 3 дней.

Ссылка: [PROJECT_URL]

Чтобы начать: откройте ссылку, войдите в аккаунт claude.ai
(бесплатного хватит), нажмите «Start new chat» и напишите
«Начните интервью».

По итогам ассистент сформирует отчёт — пожалуйста, скопируйте
его текст и пришлите нам ответным письмом. На основе отчёта
мы назначим техническое интервью с тимлидом.

Если возникнут вопросы — пишите.

С уважением,
HR-команда CodeWave Labs
```

## Часть 4. Редактирование базы знаний и промпта

### Когда что менять

| Изменение | Что править |
|---|---|
| Сменилась вакансия (Senior / Junior / другой стек) | `01_system_prompt.md` (роль, название позиции, технические вопросы), `03_vacancy_description.md`, `04_ideal_candidate.md` |
| Обновилась информация о компании (продукт, процессы, стек) | `02_company_profile.md` |
| Нужно изменить критерии оценки | `04_ideal_candidate.md` (шкала, правила) и `01_system_prompt.md` (правило итоговой рекомендации) |
| Нужно добавить новые вопросы | `01_system_prompt.md`, раздел «Структура интервью» |
| Нужно изменить формат отчёта | `01_system_prompt.md`, раздел «Формат финального резюме» |

### Процесс обновления

1. Отредактируйте нужный файл локально.
2. Откройте проект на [claude.ai](https://claude.ai).
3. В зависимости от того, что меняли:
   - **Инструкции промпта** → зайдите в **Custom instructions**, перезапишите всё содержимое, **Save**.
   - **Файл базы знаний** → в разделе **Project knowledge** удалите старую версию (иконка корзины), загрузите новую.
4. Откройте **новый** чат в проекте (старые чаты продолжают использовать предыдущие версии) и прогоните короткий тест.

### Типичные ошибки

- **Ассистент не ссылается на базу знаний.** Claude обычно читает всю базу при старте чата, но если база большая — включается выборка (RAG). В системном промпте уже указано «Ты ОБЯЗАН использовать эти документы», но при проблемах добавьте в начало первого сообщения просьбу «Сначала прочитай файлы из Project knowledge».
- **Ассистент задаёт вопросы пакетом.** Claude очень хорошо следует инструкциям, но если такое случается — усильте правило: «Один вопрос — одно сообщение. Это строгое правило. Никогда не объединяй два вопроса в одно сообщение.»
- **Отчёт формируется не в том формате.** Шаблон в промпте обёрнут в тройные бэктики — это помогает Claude точно следовать структуре.
- **В одном чате старый и новый промпт смешиваются.** После правки инструкций всегда открывайте **новый** чат для проверки — Claude помнит контекст предыдущих сообщений в рамках одного диалога.

## Часть 5. Адаптация под другую вакансию

Этот ассистент построен как шаблон. Чтобы адаптировать его под другую позицию:

1. Создайте копию проекта в Claude (вручную — через новый проект с теми же файлами и инструкциями).
2. В `01_system_prompt.md` замените:
   - Название позиции в разделе «Роль и идентичность».
   - Три технических вопроса в Этапе 2 — под нужный стек.
3. Полностью перепишите `03_vacancy_description.md` и `04_ideal_candidate.md`.
4. `02_company_profile.md` оставьте без изменений, если компания та же.
5. Пройдите тесты по чек-листу заново.

Минимальное время адаптации — 1–2 часа.

> 💡 На бесплатном тарифе Claude лимит — 5 проектов. Для большего количества вакансий нужен Pro (unlimited projects).

## Часть 6. Работа с данными кандидатов и соответствие законодательству

### Что важно учесть

Claude (Anthropic) — сторонний обработчик данных. Ответы кандидата передаются в Anthropic и обрабатываются по их политике. Это **не** закрывает требования о согласии на обработку персональных данных по GDPR / 152-ФЗ.

### Рекомендуемый порядок

1. В письме-приглашении явно укажите, что интервью проходит через сторонний сервис (Anthropic Claude) и получите согласие кандидата.
2. В системном промпте заложено правило: ассистент **не просит** и **не фиксирует в отчёте** чувствительные данные (паспорт, адрес, финансовые реквизиты).
3. Для долгосрочного хранения результатов — сохраняйте только финальный отчёт в вашу CRM / ATS, а не полную переписку.
4. На корпоративном тарифе Claude Team/Enterprise можно отключить обучение на ваших данных и подписать DPA (Data Processing Agreement) с Anthropic.
5. Если планируется массовое использование и хранение — рассмотрите переход на **Anthropic API** с размещением интерфейса в своей инфраструктуре.

### Ограничения текущей реализации

- На бесплатном тарифе нет возможности отключить использование данных для обучения моделей (на Pro и выше — можно в настройках Privacy).
- Нет автоматической выгрузки отчётов — потребуется ручное копирование кандидатом или HR.
- По общей ссылке проекта интервью может пройти кто угодно (нет встроенной идентификации).
- История диалога видна только автору чата — HR не может ретроспективно посмотреть, что именно отвечал кандидат, без копии отчёта.

Для любого из этих сценариев нужна переработка в формат **Anthropic API + собственный фронтенд**. Это следующий этап развития проекта.

## Часть 7. Отличия от реализации на OpenAI GPTs

Этот проект можно развернуть и на OpenAI GPTs (ChatGPT Plus) — структура базы знаний и промпта платформо-независимая. Основные отличия при переносе:

| Аспект | Claude Projects | OpenAI GPTs |
|---|---|---|
| Тариф для запуска | Бесплатный (до 5 проектов) | ChatGPT Plus ($20/мес) |
| Поле имени | В названии проекта | Отдельное поле Name |
| Поле описания | В описании проекта | Отдельное поле Description |
| Conversation starters | Нет, кандидат пишет сам | Настраиваемые кнопки (до 4) |
| Capabilities | Нет дополнительных тумблеров | Web Browsing / DALL·E / Code Interpreter — нужно отключить |
| Магазин | Нет публичного листинга | Есть GPTs Store |
| Шаринг | Прямая ссылка на проект | Ссылка или публичный листинг |

Все файлы из этого репозитория переносятся на OpenAI GPTs за ~10 минут — меняется только процедура настройки.
